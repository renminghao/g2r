Chapter2

seller-suggest-select

https://github.com/JedWatson/react-select 的改造版

主要功能

  • 可以代替 html 的 select
  • 支持多选
  • 支持模糊查询，本地或者接口数据都可以
  • 支持按拼音匹配
  • 支持创建自定义的option

注意

  • 这个组件因为功能完善，所以代码体积较大，如果只是需要一个简单的代替 html 的 select 的 react 组件，可以使用 mui/seller-select 。

示例

基本用法同 html select

<Select name="searchInput12" defaultValue="1">
    <Option value="1">选项1</Option>
    <Option value="2">选项2</Option>
    <Option value="3">选项3</Option>
</Select>

将options作为属性传入

假设options 的结构为：

[{
   value: 1,
   label: '标题'
}]

通过 valueKey, labelKey 指定 options 数据中对应的value 和 label

<Select name="searchInput11"
        options={options}
        defaultValue="1"
        valueKey="value"
        labelKey="label" />

支持多选

<Select multiple={true}
        name="searchInput11"
        options={options}
        defaultValue="1"
        valueKey="value"
        labelKey="label" />

支持接口模糊查询

// 接口可以直接返回数组，或者返回 { data : [] } 格式的数据
// 通过 labelKey valueKey 指定从对象的什么属性中获取 value 和 label，queryKey 用于指定查询接口的输入参数
<Select name="searchInput41"
        valueKey="value"
        labelKey="text"
        queryKey="queryKey41"
        url="http://dip.alibaba-inc.com/api/v2/services/schema/mock/16883" />

配置

Name Description Type Required Default Value
defaultValue 下拉框的默认值，变化不影响state String
value 下拉框的值 String
name 在表单中的名称 String
onChange 值变化时回调，第一个参数是value，第二个参数是option对象 Function () => {}
placeholder 没有值时显示文案 Any '请选择...'
readOnly 是否只读 Boolean false
disabled 是否禁用 Boolean false
multiple 是否多选 Boolean false
options 选项列表 Array
url 通过接口获取options数据，支持模糊查询 String
queryKey 通过接口读取数据时输入内容的接口字段名 String
valueKey options中，value字段的名字 String 'value'
labelKey options中，label字段的名字 String 'label'
matchPinyi 按照拼音匹配 Boolean true
useInitialOptions 同时使用初始化的options数据和接口返回的options数据 Boolean true
autoLoad 页面完成渲染后自动加载数据，当页面有大量select时，可以置为false，并通过options属性设置初始数据，以提高性能 Boolean true
autoLoadWhenUrlChange url变化时是否立即加载数据 Boolean false
allowCreate 允许创建自定义option Boolean false
createOptionText 创建自定义option的提示文案, "%s"为输入内容的占位符 string false '新增 "%s"'
optionsType 定义下拉选项的结构, 可选list或tree，list时是普通的列表，tree时为树状结构 String "list"
treeProps 当optionsType="tree"时用于配置tree的参数，详细内容点击这里 Obect {}

以下配置仅前端开发业务组件时需要关注

Name Description Type Required Default Value
filterOption method to filter a single option: function(option, filterString) Function no undefined
filterOptions method to filter the options array: function([options], filterString, [values]) Function no undefined
menuRenderer Renders a custom menu with options; accepts the following named parameters: menuRenderer({ focusedOption, focusOption, options, selectValue, valueArray }) Function no undefined
optionRenderer function which returns a custom way to render the options in the menu Function no undefined

更多配置可以查看原组件：https://github.com/JedWatson/react-select#further-options
